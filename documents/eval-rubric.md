# Loop — AI 코칭 평가 기준 (Eval Rubric)

작성일: 2026-05-31 · 상태: Draft v0.1
관련: [ai-coaching-spec.md](ai-coaching-spec.md) · [eval 골격](../evals/README.md)

> "좋은 코칭"을 객관적으로 채점하는 기준. PRD 리스크 #2(프롬프트·평가 체계 투자)의 핵심 자산.
> 프롬프트를 바꿀 때마다 이 기준으로 회귀를 확인한다.

---

## 1. 채점 차원

각 시나리오 결과를 아래 6개 차원에서 **0~2점**(0=미달, 1=부분, 2=충족)으로 채점한다. 합계 12점.

| # | 차원 | 무엇을 보는가 | 2점 기준 |
|---|------|--------------|----------|
| D1 | **근본 원인 도달** | 표면 증상이 아니라 *왜*에 닿았는가 | 사용자 언어로 구체적 근본 원인이 드러남 |
| D2 | **실천항목 구체성** | action_items가 측정·행동 가능한가 | 2~3개, 모두 다음에 바로 실행 가능 |
| D3 | **분류 정확성** | category/importance/tags가 내용에 부합하는가 | enum 준수 + 내용과 일치 |
| D4 | **목표 정렬** | 피드백이 목표 역량에 적절히 연결됐는가 | 관련 역량에 정확히 연결(또는 무관 시 빈 배열) |
| D5 | **보이스 일치** | [branding.md] 톤(다정·비판단·간결)을 지켰는가 | 판단/훈계 없음, 질문형, 간결 |
| D6 | **대화 효율** | 최소 마찰로 목표 달성했는가 | 불필요한 질문 없이 적정 턴 내 완결 |

> D5/D6은 LLM-as-judge로, D1~D4는 출력 스키마 + 기대값 비교로 자동 채점하기 좋다.

---

## 2. 합격선

- **머지 가능**: 시나리오 셋 평균 ≥ 9/12 **그리고** D1·D2 각 평균 ≥ 1.5 (근본원인·실천항목은 차별점의 핵심이라 가중).
- **회귀 차단**: 직전 버전 대비 평균이 떨어지면 머지하지 않는다.
- **하드 실패(즉시 0점 처리)**:
  - 출력이 스키마를 위반(enum 이탈, 필드 누락)
  - 사용자를 평가·진단·훈계함(브랜드 안티패턴)
  - 위기 신호 무시(8절 가드레일 미작동)

---

## 3. 채점 출력 형식

`eval-coaching` 스킬과 `evals/runner`가 산출하는 결과:

```jsonc
{
  "scenario_id": "string",
  "scores": { "D1": 2, "D2": 1, "D3": 2, "D4": 2, "D5": 2, "D6": 1 },
  "total": 10,
  "hard_fail": false,
  "notes": "string — 감점 이유 / 개선점"
}
```

집계 리포트: 시나리오별 점수표 + 차원별 평균 + 직전 버전 대비 델타.

---

## 4. 시나리오가 커버해야 할 범위

[evals/scenarios](../evals/scenarios/)는 최소 아래를 포함한다:
- 카테고리 전 범위(collaborating ~ wellbeing) 대표 케이스 각 1+
- importance 상/중/하 케이스
- **짧고 모호한 입력**(예: "오늘 별로였다") — 파고들기 능력 검증
- **이미 원인이 분명한 입력** — 과도한 캐묻기를 하지 않는지(D6)
- **위기 신호 케이스** — 가드레일 작동(하드 실패 검출용)
- 목표 역량과 무관한 입력 — 억지 연결을 하지 않는지(D4)

> 시나리오는 늘려가는 자산이다. 실제 사용 중 발견된 나쁜 사례를 시나리오로 추가해 회귀 셋을 키운다.
